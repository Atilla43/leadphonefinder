# Логика поведения AI-продажника

## Статусы получателей

| Статус | Описание |
|--------|----------|
| `pending` | Начальное состояние, ещё не контактировали |
| `sent` | Первое сообщение отправлено, ждём ответ |
| `talking` | Активный диалог, интерес пока не проявлен |
| `warm` | Лид проявил интерес (спросил цену/сроки, согласился на звонок) |
| `warm_confirmed` | Тёплый лид ответил после предложения созвона |
| `rejected` | 3+ явных отказа |
| `referral` | Лид перенаправил на другого человека |
| `no_response` | Исчерпаны follow-up попытки (2 пинга без ответа) |
| `not_found` | Номер не найден в Telegram |
| `error` | Техническая ошибка отправки |

## Статусы кампании

| Статус | Описание |
|--------|----------|
| `pending` | Не запущена |
| `sending` | Фаза 1: рассылка первых сообщений |
| `listening` | Фаза 2: слушаем ответы, AI отвечает |
| `paused` | Поставлена на паузу пользователем |
| `completed` | Все сообщения обработаны |
| `cancelled` | Отменена пользователем |

---

## Фаза 1: Рассылка первых сообщений (`start_campaign`)

1. **Распределение** — получатели распределяются по аккаунтам round-robin
2. **Рабочие часы** — отправка только 10:00–17:00 МСК
3. **Дневной лимит** — при достижении лимита ждёт до следующего дня 10:00 МСК
4. **Для каждого `pending` получателя:**
   - Импорт контакта через Telethon (`ImportContactsRequest`)
   - Если не в Telegram → `not_found`
   - Отправка первого сообщения: `"{Имя}, здравствуйте, коротко по «{Компания}».\n\n{Оффер}"`
   - Статус → `sent`
   - Сообщение добавляется в `conversation_history`
5. **Интервал между отправками:**
   - Формула: `оставшееся_время / кол-во_отправок`
   - Растягивает рассылку на весь рабочий день
   - Джиттер ±2 мин, минимум 30 сек
6. **FloodWait** — при ограничении Telegram ждёт указанное время и повторяет
7. **Listener запускается ПАРАЛЛЕЛЬНО с рассылкой** — ответы обрабатываются сразу

## Фаза 2: Слушатель ответов (`start_listener`)

### Механизм дебаунса (30 сек)

Предотвращает множественные AI-вызовы когда лид шлёт несколько сообщений подряд:

1. Сообщение 1 → добавляется в `_pending_messages[sender_id]`, таймер 30 сек
2. Сообщение 2 (в течение 30 сек) → добавляется, таймер сбрасывается
3. 30 сек без сообщений → все сообщения объединяются и обрабатываются как одно

### Обработка контактных карточек

Если лид присылает визитку — текст формируется как `[Контакт: имя, телефон]`

### Умная отмена во время набора

Пока бот "печатает" (typing indicator), каждые 3 сек проверяет: пришли ли новые сообщения?
- Если да → отменяет отправку, новый дебаунс подхватит всё
- Если нет → отправляет сообщение

## Обработка по статусам

### Статус: `sent` или `talking`

Обычный диалог:

1. Сообщения лида добавляются в `conversation_history`
2. `ping_count` сбрасывается в 0 (лид ответил — пинги не нужны)
3. AI генерирует ответ (3 попытки, таймаут 60 сек каждая)
   - Использует: `system_prompt`, `company_context` (данные о компании)
   - LLM возвращает: `{"reply": "текст", "status": "talking|warm|rejected|referral"}`
4. Имитация набора текста (4–45 сек в зависимости от длины)
5. **По статусу от AI:**
   - `talking` → статус остаётся `talking`
   - `warm` → статус = `warm`, уведомление пользователю
   - `rejected` → статус = `rejected`
   - `referral` → статус = `referral`, запуск обработки referral
6. При провале AI после 3 попыток → повтор через 60 сек (`_retry_failed_ai`)

### Статус: `warm`

Лид уже проявил интерес, отвечает на предложение созвона:

1. Сообщение добавляется в `conversation_history`
2. AI генерирует ответ (3 попытки, таймаут 60 сек)
3. Отправка с имитацией набора
4. Статус → `warm_confirmed`
5. Уведомление пользователю
6. При провале AI → повтор через 60 сек

### Статус: `warm_confirmed`

Лид уже подтверждённый тёплый:

1. Сообщение сохраняется в `conversation_history`
2. Уведомление пользователю о новом сообщении
3. **AI НЕ отвечает** — дальше работает человек
4. Проверка на referral: если в сообщении есть телефонные паттерны (`+7`, `89`, `Контакт:`) → запуск `_check_warm_referral`

### Статус: `referral`

Лид перенаправил на другого человека, ждём контакт:

1. Сообщение добавляется в `conversation_history`
2. Вызов `_handle_referral()` — поиск телефона/имени во всей переписке
3. Если контакт найден → обработка
4. Если нет → ждём дальнейших сообщений

### Статусы: `rejected`, `no_response`

Сообщения **игнорируются** — бот не реагирует.

## Referral: обработка перенаправлений

Когда AI определяет статус `referral` или warm_confirmed лид даёт контакт:

1. **Извлечение контакта через AI** (`extract_referral_contact`, таймаут 30 сек)
   - Промпт `REFERRAL_EXTRACT_PROMPT` ищет телефон и имя
   - Возвращает: `{"phone": "+7XXXXXXXXXX", "name": "Имя"}`

2. **Проверка дубликатов** — если телефон уже в кампании → пропуск

3. **Создание нового получателя:**
   - Копирует: категорию, адрес, аккаунт от оригинального лида
   - `referral_context` = выжимка из оригинальной переписки

4. **Импорт контакта** через Telethon
   - Если не в Telegram → `not_found`

5. **Генерация первого сообщения через AI** (`generate_referral_first_message`, таймаут 30 сек)
   - Промпт: использовать контекст переписки, не писать как холодному лиду
   - Приветствие через "Здравствуйте"
   - Fallback шаблон если AI не справился

6. **Отправка и трекинг:**
   - Статус = `sent`
   - Добавление в `campaign.recipients`
   - Уведомление пользователю

## Follow-up / Пинги (`_ping_loop`)

Фоновый цикл, проверяет каждые 60 минут (только 10:00–17:00 МСК):

| День | ping_count | Действие |
|------|------------|----------|
| День 1 | 0 | Ничего — ждём ответа |
| День 2 | 0 → 1 | Сообщение: `"{Имя}, подниму сообщение)) возможно, потерялось"` (+ 🌷 для женщин) |
| День 3 | 1 → 2 | Отправка стикера из настроенного пака |
| День 4+ | ≥2 | Статус → `no_response`, пинги прекращаются |

**Сброс пингов:** когда лид отвечает, `ping_count` сбрасывается в 0.

**Определение пола:** по отчеству — `-овна`/`-евна` = женский, `-ович`/`-евич` = мужской.

## Retry-механизмы

### При рестарте бота (`retry_unanswered`)

1. Проверяет всех получателей кампании
2. **warm_confirmed** с телефонными паттернами в переписке → `_check_warm_referral`
3. Последнее сообщение от `user` (AI не ответил) → `_process_unanswered`
   - 3 попытки AI, таймаут 60 сек
   - Отправка ответа
   - Обновление статуса

### При провале AI в реальном времени (`_retry_failed_ai`)

1. Ждёт 60 секунд
2. Проверяет что сообщение всё ещё без ответа
3. Повторно вызывает `_process_recipient_messages`

## Уведомления пользователю

| Событие | Когда |
|---------|-------|
| `warm_lead` | Лид стал тёплым |
| `warm_lead_reply` | Тёплый/подтверждённый лид написал |
| `referral` | Найден и обработан referral-контакт |
| `sending_complete` | Все первые сообщения отправлены |
| `daily_limit` | Достигнут дневной лимит |
| `flood_wait` | Telegram ограничение, пауза |

Уведомления отправляются владельцу кампании и всем менеджерам (`manager_ids`).

**Важно:** после рестарта бота notify callback восстанавливается в `main.py` — уведомления работают и после перезапуска.

## Контекст для AI

К каждому AI-вызову прикладывается `company_context`:

- Категория компании
- Рейтинг, сайт, часы работы, адрес
- ФИО директора
- Для referral: `"ЭТОТ КОНТАКТ — ПЕРЕНАПРАВЛЕНИЕ"` + история оригинальной переписки

## AI-промпты

### SALES_SYSTEM_PROMPT (основной диалог)
- Персонаж: Александр, менеджер
- Цель: заинтересовать → ответить на вопросы → созвон
- Правила: не перескакивать этапы, отвечать по контексту
- Запреты: не признавать что AI, грамотный русский
- Статусы: `talking` / `warm` / `rejected` / `referral`

### REFERRAL_FIRST_MSG_PROMPT (первое сообщение referral)
- Использовать контекст переписки
- Здравствуйте (не Привет)
- Не писать как холодному лиду

### REFERRAL_EXTRACT_PROMPT (извлечение контакта)
- Найти телефон и имя из переписки
- Формат: `+7XXXXXXXXXX`

### FOLLOWUP_SYSTEM_PROMPT (follow-up при игноре)
- Короткий крючок: вопрос, мини-кейс, факт
- Не повторять предыдущее

## Диаграмма переходов статусов

```
pending
  │
  ├─ телефон не найден ──→ not_found
  ├─ ошибка отправки ───→ error
  └─ сообщение отправлено → sent
                              │
                              ├─ 2 пинга без ответа ──→ no_response
                              └─ лид ответил ─────────→ talking
                                                          │
                                                          ├─ AI: talking ──→ talking (продолжаем)
                                                          ├─ AI: warm ────→ warm
                                                          │                   │
                                                          │                   └─ лид ответил → warm_confirmed
                                                          │                                      │
                                                          │                                      └─ дал контакт → referral обработка
                                                          ├─ AI: rejected ─→ rejected
                                                          └─ AI: referral ─→ referral
                                                                              │
                                                                              ├─ контакт найден → новый recipient (sent)
                                                                              └─ ждём контакт
```
